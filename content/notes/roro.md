## Summary

[roro](/notes/roro/) is a platform for managing multiple AI coding agents working simultaneously on coding tasks. Each project is a workspace; you add agents to each workspace and assign them tasks. The distinctive feature is the Agent Calendar — described as \"the first calendar not for you, but for your agents\" — for scheduling tasks to run at night, complementing [Nightshift](/notes/nightshift/)'s approach to using off-hours agent capacity.

Real-time activity monitoring comes through automatically configured Claude Code hooks, so you see what agents are doing without custom setup. Agents can be personalized with names, emojis, and profile pictures — \assemble your dream team.\ The scratchpad feature provides quick access to notes and ideas shared across the workspace.

This is the same design territory as Baton, [dmux](/notes/dmux/), [cmux](/notes/cmux/), and [The Companion](/notes/the-companion/) — tools for managing multiple AI agents in parallel without losing track of what each one is doing. [roro](/notes/roro/) distinguishes itself with the scheduling/calendar angle and the agent personality features, targeting developers who want a persistent multi-agent environment rather than one-off parallel runs.

## Key points

- Workspace-per-project with multiple agents per workspace.
- Agent Calendar: schedule tasks for overnight execution.
- Real-time activity via auto-configured Claude Code hooks.
- Agent personalization (name, emoji, avatar) for identity in multi-agent teams.
- Scratchpad for workspace-level notes.
- Complements [Nightshift](/notes/nightshift/) (scheduled maintenance) with more general parallel scheduling.
