## Summary

This post describes a local-first writing workflow using Obsidian for drafting and a publishing pipeline that converts vault notes to live blog posts. The vault lives in iCloud (or Dropbox/Google Drive), enabling seamless cross-device drafting — start a note on the phone during a commute, finish it on the laptop, publish without leaving the tool.

The appeal of Obsidian for writing is the plain-text-on-disk model: notes are Markdown files you own, readable without the app, not locked into any cloud service. The Minimal theme keeps the interface clean and distraction-free, which the author finds better for writing flow than editor-heavy alternatives. The key workflow insight is that the vault-to-blog conversion step can be automated, making write note → publish post a single action.

This workflow connects to the broader local-first software movement: data lives on your machine first, sync is a secondary concern, and the tool respects the user's ownership of their content. For writers using Obsidian as a knowledge base, having the blog be a subset of the vault makes thematic connections between posts visible as graph edges.

## Key points

- Obsidian vault stored in iCloud for cross-device sync — draft on phone, publish from laptop.
- Minimal theme: clean, distraction-free writing environment.
- Automated vault → blog publishing pipeline — single action to go from note to live post.
- Local-first software: files are plain Markdown on disk, not cloud-locked.
- Blog posts as a subset of the vault: interlinking is natural.
- Related to Hugo, Jekyll, and other static site generators that consume Markdown.
