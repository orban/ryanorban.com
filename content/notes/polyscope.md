## Summary

[Polyscope](/notes/polyscope/) positions itself as "the new cockpit" for software development — a central control interface designed for the era where AI writes code and humans supervise and direct rather than type line by line. The tagline: "The time where humans write code is over."

The cockpit metaphor is deliberate. A pilot doesn't fly by manually moving every control surface — they set direction, monitor instruments, and intervene when the automated systems need guidance. [Polyscope](/notes/polyscope/) maps this to software development: you're not writing code keystroke by keystroke, you're directing AI agents and reviewing their output from a unified interface.

This framing positions it alongside Baton, [cmux](/notes/cmux/), and Ona in the space of tools that assume multi-agent development is the default and build interfaces for it. Limited public detail is available, but the positioning is part of an emerging category: developer interfaces designed around agent supervision rather than manual coding.

## Key points

- "The new cockpit" metaphor: humans as supervisors directing AI, not manually writing every line.
- Positioned in the emerging category of agent-native developer interfaces.
- Aligns with Baton, [cmux](/notes/cmux/), and similar multi-agent dashboard tools.
- Limited public detail available; notable primarily for its framing of the human role in AI-driven development.
