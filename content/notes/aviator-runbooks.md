## Summary

[Aviator Runbooks](/notes/aviator-runbooks/) positions AI coding as a collaborative team activity rather than an individual developer tool. The concept: instead of each developer maintaining their own AI agent configuration and prompts, teams create shared runbooks — reusable AI task specifications that anyone on the team can execute. The pitch is turn AI-coding multiplayer.

The runbooks serve multiple functions: automating PR reviews (consistent standards applied by AI rather than varying by reviewer), standardizing common workflows (how the team handles migrations, refactors, dependency upgrades), and integrating with existing AI tools. The reusable part is key — a runbook written once for a specific task type becomes team infrastructure rather than throwaway prompt engineering.

This connects to spec-driven development patterns like [Zenflow](/notes/zenflow/) and [Augment Intent](/notes/augment-intent/), but oriented around team process rather than individual agent orchestration. The focus on PR review automation is practical: code review is one of the most time-consuming parts of software development, and inconsistency between reviewers is a common frustration.

## Key points

- Shared runbooks that any team member can execute — AI coding as team infrastructure, not individual tooling.
- Automates PR reviews with consistent, shareable AI standards.
- Reusable: a runbook defined once for a task type serves every future instance.
- Integrates with existing AI coding agent tools rather than replacing them.
- Positions as the collaboration and standardization layer on top of AI coding.
