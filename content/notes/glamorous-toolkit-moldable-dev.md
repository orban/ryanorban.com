## Summary

Glamorous Toolkit (GT) is a development environment built around the concept of moldable development — the idea that developer tools should be customized for each system you're working on, rather than being generic one-size-fits-all tools. In GT, you can add custom views, inspectors, and debuggers for specific domain objects, turning the IDE itself into a knowledge base about your system.

Built on Pharo (a Smalltalk dialect), GT takes Smalltalk's live programming tradition and extends it with a focus on explainability. The environment features Lepiter (its live notebook system), custom object inspectors, and a browser that's as easy to extend as the rest of the system. Unlike traditional IDEs where you read documentation about your system, in GT you build interactive visualizations and tools that make the system explain itself.

The moldable development philosophy sits in an interesting niche: it requires more investment upfront (you're essentially building tools alongside your system), but claims to make the system more understandable over time. It's particularly well-suited to complex domain models where understanding the relationships and state of objects is the primary challenge. The Pharo foundation gives it access to decades of live programming culture.

## Key points

- Moldable development: developer tools are first-class artifacts to be built alongside the system.
- Custom views and inspectors per domain object — system-specific IDE extensions are the norm.
- Built on Pharo / Smalltalk — live programming, image-based environment.
- Lepiter notebook system: live documentation that stays in sync with the system.
- Shifts IDE from generic editor to system-specific knowledge tool.
- Particularly suited to complex domain models and systems where explainability is the challenge.
